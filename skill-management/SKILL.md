# Skill Management

설치된 스킬 전체를 관리하는 통합 관리 스킬. 4가지 모드로 동작한다.

## 스킬 경로

- 사용자 스킬: `/mnt/skills/user/`
- 공개 스킬: `/mnt/skills/public/`
- 예시 스킬: `/mnt/skills/examples/`

---

## 모드 1: 카탈로그 (목록 조회)

트리거: "내 스킬 보여줘", "스킬 목록", "어떤 스킬 있어"

### 실행 방법

1. `/mnt/skills/` 하위 디렉토리를 스캔해 각 스킬의 `SKILL.md` frontmatter를 읽는다
2. 각 스킬의 `name`과 `description`을 수집한다
3. description 내용을 바탕으로 비슷한 스킬끼리 카테고리를 직접 만들어 분류한다
   - 카테고리 이름과 수는 설치된 스킬 구성에 따라 매번 달라진다
   - 어디에도 속하지 않는 스킬은 기타로 묶는다
4. 카테고리별로 스킬 목록을 출력한다

### 출력 형식

```
[카테고리 이모지] [카테고리명] (N개)
  • 스킬명 — description 첫 문장
  ...
```

## 모드 2: 중복/충돌 감지

트리거: "스킬 중복 확인", "겹치는 스킬 있어", "스킬 정리"

### 실행 방법

1. `/mnt/skills/user/`, `/mnt/skills/public/` 의 모든 SKILL.md를 스캔해 name과 description을 수집한다
2. description에서 핵심 기능 키워드를 추출한다
3. 모든 스킬 쌍을 비교해 description의 목적과 출력이 유사한 그룹을 찾는다
4. 유사 그룹별로 실제 역할 차이를 분석해 "진짜 중복"인지 "분업 구조"인지 판단한다

### 판단 기준

- **진짜 중복**: 동일한 입력에 동일한 출력을 내는 스킬. 새 스킬로 통합하거나 하나를 제거 권장
- **분업 구조**: 같은 도메인이지만 입력 조건이나 출력 포맷이 명확히 다른 스킬. 유지 권장
- **브랜드 전용 vs 범용**: 특정 브랜드/프로젝트 전용 스킬은 범용 스킬과 별도 취급

### 출력 형식

각 유사 그룹별로:
- 스킬 이름 목록
- 공통점과 차이점 한 줄 요약
- 판단: 유지 / 새 스킬로 통합 고려 / 사용자 결정 필요

## 모드 3: 스킬 추천

트리거: "X 작업에 어떤 스킬 써야 해?", "~할 때 스킬 뭐 써", "스킬 추천"

### 실행 방법

1. `/mnt/skills/user/`, `/mnt/skills/public/` 의 모든 SKILL.md를 스캔해 name과 description을 수집한다
2. 사용자의 작업 설명에서 핵심 키워드를 추출한다 (결과물 형식, 도구, 플랫폼, 시작 조건 등)
3. 각 스킬의 description과 키워드를 매칭해 후보를 좁힌다
4. 후보가 여럿이면 아래 기준으로 최종 1~2개를 선택한다

### 동일 도메인 내 선택 기준

같은 카테고리 안에서 여러 스킬이 후보일 때:

- **결과물 형식** — standalone HTML인지, React 컴포넌트인지, JSON인지, 마크다운인지
- **시작 조건** — 이미지가 있는지, 텍스트만 있는지, 클라이언트 브리핑이 필요한지
- **플랫폼/도구** — 특정 플랫폼 전용인지, 특정 브랜드 전용인지, 범용인지
- **작업 단계** — 처음 만드는 건지, 기존 것을 개선하는 건지

### 출력 형식

추천 스킬명과 함께 "왜 이 스킬인지" 한 줄 이유를 반드시 제시한다.
후보가 2개 이상일 경우 각각의 적합한 상황을 구분해서 설명한다.

## 모드 4: 스킬 진단

트리거: "스킬 진단", "스킬 점수 매겨줘", "스킬 품질 확인"

### 점검 항목

각 스킬의 SKILL.md를 읽고 아래 기준으로 점수를 매긴다:

| 항목 | 기준 | 배점 |
|------|------|------|
| 트리거 명확성 | description에 구체적 트리거 문구 포함 | 30점 |
| 설명 길이 | 50~300자 (너무 짧거나 길면 감점) | 20점 |
| 한국어/영어 일관성 | 스킬 주요 언어와 description 일치 | 20점 |
| 예시 포함 | 사용 예시나 트리거 예문 포함 | 30점 |

### 출력 형식

점수 리포트 → 공통 문제 패턴 요약 순으로 출력한다.
반드시 "진단 완료." 로 끝낸다. 그 뒤에 어떤 말도 덧붙이지 않는다.

```
📊 스킬 진단 리포트

✅ 양호 (80점 이상): ...
⚠️ 개선 권장 (50~79점): ...
  → 스킬명: 문제점 한 줄 요약
❌ 주의 (49점 이하): ...
  → 스킬명: 문제점 한 줄 요약

🔍 공통 패턴: (있을 경우에만)
  → 패턴 요약

진단 완료.
```

### 절대 원칙

리포트 어디에도 수정, 개선, 보강, 업데이트, 고치기에 관한 언급을 하지 않는다.
문제점 설명은 사실 나열로만 끝낸다.

---

## 사용 흐름

사용자의 요청이 모드를 명확히 지정하지 않은 경우:
1. "내 스킬 어떻게 돼 있어?" 같은 포괄적 질문 → 모드 1(카탈로그) 실행 후 다른 모드 제안
2. 작업 언급 없이 "정리해줘" → 모드 2(중복 감지) + 모드 4(진단) 동시 실행
3. 특정 작업 언급 → 모드 3(추천) 실행
